동작 방식
1) Source 클릭해서 이미지 업로드
2) 1차 프롬프트 입력 후 1차 Make
3) 필요하면 Reference 업로드
4) 2차 프롬프트 입력 후 2차 Make
5) 2차 결과가 Final Viewer에 자동 반영
참고: 백엔드 /api/render가 없으면 데모 모드로 입력 이미지를 그대로 넘겨서 흐름만 확인하게 돼.
